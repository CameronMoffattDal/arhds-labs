Introduction

This practical explores natural language processing (NLP) of unstructured clinical free-text from electronic medical records (EMR). You will work through an analysis going from raw text to an unsupervised topic model using the MTSamples medical transcription dataset.

The analyses you will do mirror some of those taken in the assigned readings and refer back to them for some questions:

  • Sun et al. (2024) (Latent Dirichlet Allocation (LDA)-based topic modelling for clinical social work notes)
  • Chung et al. (2024) (GPT-4 Turbo for classifying perioperative outcomes from pre-operative notes).

Learning objectives: by the end of this practical you should understand how to:

  • apply regular expressions to search clinical free-text
  • tokenise, normalise, stem, and lemmatize medical text
  • compute TF-IDF to identify specialty-discriminating terms
  • build a document-term matrix and fit an LDA topic model on clinical notes
  • compare traditional NLP with LLM-based approaches

Regular Expressions

Now we are going to learn how to use the stringr library for regular expressions in R. Regular expressions (regex) enable flexible pattern-based searches over text. As discussed in the lecture, they are widely used in NLP (including as part of the framework around LLM systems) for finding measurements, medications, and semi- structured values embedded in free-text.

Key stringr functions:

str_detect(x, pat) Returns TRUE/FALSE per string
str_count(x, pat) Number of non-overlapping matches
str_extract(x, pat) First match
str_extract_all(x, pat) All matches (returns a list)

Finding blood pressure measurements

Blood pressure is typically recorded as systolic/diastolic (e.g., “120/80” or “130/85 mmHg”). We can therefore look for a pair of 2-3 digit numbers (\\d{2,3}) separated by a / (i.e., \\d{2,3}/\\d{2,3}) optionally followed by a pressure-related mmHg unit (with or without a preceding space \\s*). The special regex characters like \\d and \\s have that extra \ so they are “escaped” and not treated as a literal character.

bp_pattern <- "\\d{2,3}/\\d{2,3}(\\s*mmHg)?"


filtered_data |>
  mutate(has_bp = str_detect(transcription, bp_pattern)) |>
  count(medical_specialty, has_bp) |>
  pivot_wider(names_from = has_bp, values_from = n, names_prefix = "bp_") |>
  mutate(pct_with_bp = percent(bp_TRUE / (bp_TRUE + bp_FALSE), accuracy = 0.1))

Text Normalisation

A key stage in traditional text analysis is to normalise raw text — reducing variability due to capitalisation, punctuation, and common function words — before any modelling or analysis.

Tokenisation and stop word removal

tidytext::unnest_tokens() splits text into individual tokens and lowercases everything. We then strip residual punctuation, drop very short tokens, and remove stop words (common words like “the”, “of”, “and” that carry little domain information):

tokenised_data <- filtered_data |>
  unnest_tokens(word, transcription, to_lower = TRUE) |>
  mutate(word = str_replace_all(word, "[^[:alpha:]]", "")) |>
  filter(str_length(word) > 1) |>
  anti_join(stop_words, by = "word")

Bigrams

We can also combined consecutive words into two-word phrases (bigrams) which better capture medical compound terms that unigrams split apart (e.g., “blood pressure”, “right knee”, “general anaesthesia”):

bigrams <- filtered_data |>
  unnest_tokens(
    bigram,
    transcription,
    token = "ngrams",
    n = 2,
    to_lower = TRUE
  ) |>
  filter(!is.na(bigram)) |>
  mutate(word1 = word(bigram, 1), word2 = word(bigram, 2)) |>
  filter(
    !word1 %in% stop_words$word,
    !word2 %in% stop_words$word,
    !str_detect(word1, "\\d"),
    !str_detect(word2, "\\d")
  ) |>
  select(-word1, -word2)

b <- bigrams |>

Stemming and Lemmatization

However, given how noisy language is and the many variants of the same word (alternative suffixes) like pluralisation or possessive apostrophes. We also often want to “normalise” words to a common base form. There are two main approaches to this:

  • Stemming-crude suffix stripping, fast but imprecise: “operating” to “operat” (SnowballC::wordStem)

  • Lemmatization-linguistically informed root form: “operating” to “operate” (textstem::lemmatize_words)

    We can apply these normalisations using the associated library functions.

word_sample <- tokenised_data |>
  distinct(word) |>
  mutate(
    stem = SnowballC::wordStem(word, language = "english"),
    lemma = textstem::lemmatize_words(word)

TF-IDF: Finding Discriminating Terms

Unfortunately, as you can see above, raw counts are dominated by generic medical words (“patient”, “history”, “left”). TF-IDF (term frequency–inverse document frequency) down-weights terms common across all documents and up-weights terms characteristic of a specific category. Sun et al. (2024) used word-frequency analysis as a first step before LDA to explore the vocabulary of their social work notes.

specialty_tfidf <- lemmatised_data |>
  count(medical_specialty, lemma) |>
  bind_tf_idf(lemma, medical_specialty, n)

specialty_tfidf |>
  group_by(medical_specialty) |>
  slice_max(tf_idf, n = 25, with_ties = FALSE) |>
  ggplot(aes(
    tf_idf,
    reorder_within(lemma, tf_idf, medical_specialty),
    fill = medical_specialty
  )) +
  geom_col(show.legend = FALSE) +
  facet_wrap(~medical_specialty, scales = "free") +

Topic Modelling with Latent Dirichlet Allocation

Latent Dirichlet Allocation (LDA) is an unsupervised probabilistic model that treats each document as a mixture of topics and each topic as a probability distribution over words. Unlike supervised classifiers, LDA discovers thematic structure from the text alone-no specialty labels are used during training. This mirrors the approach of Sun et al. (2024), who applied LDA to clinical social work notes to uncover latent SDoH themes without predefined categories.

Two key outputs of a fitted LDA model:

Output Symbol Interpretation
Word–topic matrix β (beta) Probability that word w is generated by topic k
Document–topic matrix γ (gamma) Proportion of document i.e., transcript d attributed to topic k

Building the document-term matrix

We aggregate lemma counts per note and apply two filters before building the DTM:

  1. Minimum document frequency - drop terms appearing in fewer than 5 notes to reduce sparsity.
  2. TF-IDF filtering - drop terms whose mean TF-IDF (computed per note) falls in the bottom quartile. Words that are common across nearly all notes have low IDF and therefore low TF-IDF; retaining them would give LDA noisy, non-discriminative signal.
term_counts <- lemmatised_data |>
  count(note_id, lemma)

# Keep only terms present in >= 5 notes
doc_freq_filter <- term_counts |>
  distinct(note_id, lemma) |>
  count(lemma, name = "doc_count") |>
  filter(doc_count >= 5)

# Keep only terms whose mean per-note TF-IDF is above the 25th percentile
tfidf_filter <- term_counts |>
  bind_tf_idf(lemma, note_id, n) |>
  group_by(lemma) |>
  summarise(mean_tfidf = mean(tf_idf), .groups = "drop") |>
  filter(mean_tfidf > quantile(mean_tfidf, 0.25))

emr_dtm <- term_counts |>
  semi_join(doc_freq_filter, by = "lemma") |>
  semi_join(tfidf_filter, by = "lemma") |>
  cast_dtm(note_id, lemma, n)

emr_dtm <- emr_dtm[rowSums(as.matrix(emr_dtm)) > 0, ]

cat("DTM:", nrow(emr_dtm), "documents ×", ncol(emr_dtm), "terms\n")
DTM: 851 documents × 2518 terms

Fitting the LDA model

We fit LDA with k = 3 topics-one per specialty-so we can directly compare the discovered topics against the known specialty labels. Again this would likely be something you explore or optimise using more sophisticated hierarchical models.

emr_lda <- topicmodels::LDA(emr_dtm, k = k_topics, control = list(seed = 42))
